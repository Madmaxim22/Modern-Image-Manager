# Modern-Image-Manager

![Test and Deploy](https://github.com/Madmaxim22/Modern-Image-Manager/workflows/Test%20and%20Deploy/badge.svg)
[![GitHub Pages](https://img.shields.io/badge/GitHub-Pages-blue?logo=github)](https://madmaxim22.github.io/Modern-Image-Manager/)

Современный веб-инструмент для управления изображениями с поддержкой drag-and-drop. Приложение позволяет легко загружать изображения в галерею как через перетаскивание файлов, так и через стандартный диалог выбора файлов. После загрузки изображения можно просматривать и удалять при необходимости. Все изображения хранятся на сервере.

## Особенности

- **Drag-and-drop интерфейс**: Перетащите изображения прямо в зону загрузки
- **Поддержка нескольких файлов**: Загружайте несколько изображений за один раз
- **Управление изображениями**: Просмотр и удаление загруженных изображений
- **Сохранение на сервере**: Изображения сохраняются на сервере и доступны при перезагрузке страницы
- **Адаптивный дизайн**: Удобное использование на различных устройствах
- **Поддержка основных форматов изображений**: JPEG, PNG, GIF, WebP и другие
- **Корректная обработка имен файлов**: Поддержка кириллических и специальных символов в именах файлов
- **Ведение логов**: Детализированные логи операций на сервере
- **Клиентский API сервис**: Централизованная обработка запросов к серверу

## Технологии

- **HTML5**: Семантическая верстка и API для работы с файлами
- **CSS3**: Современные стили и адаптивный дизайн
- **JavaScript (ES6+)**: Объектно-ориентированная архитектура и функциональность
- **Express.js**: Серверная часть для обработки запросов
- **Multer**: Обработка загрузки файлов
- **Webpack**: Сборка проекта и управление зависимостями
- **Babel**: Поддержка современных возможностей JavaScript
- **ESLint**: Контроль качества кода
- **UUID**: Генерация уникальных идентификаторов файлов

## Установка

1. Клонируйте репозиторий:
```bash
git clone https://github.com/Madmaxim22/Modern-Image-Manager.git
```

2. Перейдите в директорию проекта:
```bash
cd Modern-Image-Manager
```

3. Установите зависимости для frontend и backend:
```bash
npm install
cd backend && npm install && cd ..
```

## Использование

### Запуск сервера

Для запуска backend сервера:
```bash
cd backend && npm start
```

Сервер будет запущен на [http://localhost:7070](http://localhost:7070)

### Запуск в режиме разработки

Для запуска frontend приложения в режиме разработки:
```bash
npm start
```

Приложение будет доступно по адресу [http://localhost:5000](http://localhost:5000)

### Сборка для продакшена

Для создания оптимизированной сборки проекта:

```bash
npm run build:prod
```

Собранные файлы будут находиться в папке `build/`.

### Разработка

Для сборки проекта в режиме разработки:

```bash
npm run build:dev
```

## Функциональность

1. **Загрузка изображений**:
   - Перетащите изображения в зону загрузки
   - Кликните по зоне загрузки для выбора файлов через диалог
   - Поддержка множественного выбора файлов
   - Изображения загружаются на сервер и сохраняются
   - Поддержка имен файлов с кириллическими и специальными символами

2. **Просмотр изображений**:
   - Загруженные изображения отображаются в галерее
   - Изображения автоматически масштабируются для оптимального отображения
   - При перезагрузке страницы изображения подгружаются сервера

3. **Управление изображениями**:
   - Удаление отдельных изображений с помощью кнопки "×"
   - При удалении изображения удаляются как с галереи, так и с сервера
   - Визуальная обратная связь при взаимодействии

4. **Валидация**:
   - Проверка формата файла (только изображения)
   - Отображение сообщений об ошибках

5. **Логирование**:
   - Детализированные логи операций на сервере (загрузка, удаление, получение файлов)
   - Логирование времени и типа операции

## API endpoints

- `GET /files` - получить список всех загруженных файлов
- `GET /files/:id` - получить информацию о конкретном файле
- `POST /files` - загрузить новый файл
- `DELETE /files/:id` - удалить файл
